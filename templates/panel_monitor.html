Revisiones de Notas

Acceso limitado detectado

Para modificar estados academicos (ej. /api/academic/update) no puedes hacerlo directamente. Debes cumplir estos requisitos:

  • Sesion activa: Tienes que estar logueado. La cookie session_data debe tener tu user_id.
  • Token de coordinador: Necesitas un token valido de coordinador sin restricciones y guardarlo como cookie access_token.
  • Obtener el token: El endpoint de verificacion es POST /api/verify-token. Envia el token en el header Authorization: Bearer <token>. El token debe ser Base64 de un JSON con role: "coordinator" y limited: false.
  • En esta pagina (panel de monitor) puedes encontrar en el codigo fuente una forma de generar y verificar ese token. Revisa el HTML y el JavaScript.

Resumen: inicia sesion, llega a este panel, obten el token (revisa el codigo fuente), guardalo como cookie access_token, y entonces las peticiones a /api/academic/update con tu sesion activa funcionaran.
